# ToDo アプリ 要件定義

## コンセプト

「今日やることだけに集中する」シンプルな日次タスク管理アプリ。
DayTask (https://apps.apple.com/jp/app/daytask-seize-your-day/id6760099687) を参考にしつつ、iPhone / Mac で同じデータを共有できるようにする。

---

## ユーザー

- 利用者は自分1人のみ
- ログイン機能は不要
- 固定シークレット（Bearer トークン）による簡易保護

---

## 機能要件

### タスク管理（CRUD）

- タスクを追加できる（右下の「+」ボタン）
- タスクを完了 / 未完了に切り替えできる（チェックボックス）
- タスクを削除できる
- タスクの並び順をドラッグ&ドロップで変更できる
- 完了済みタスクは打ち消し線 + グレーアウトで表示し、リストの下部にまとめる

### 日付ナビゲーション

- メイン画面は「今日のタスク」を表示する
- 画面上部に日付ヘッダーを表示（例: "Today" / "Yesterday" + "2026-04-01 (Wed)"）
- iOS: 横スワイプで日付を移動（矢印ボタンなし）
- macOS: 左右の矢印ボタンで日付を移動
- 今日より未来には進めない

### 日付ごとの編集制限

- **今日**: タスクの追加・編集・削除・並べ替えがすべて可能
- **1日前（Yesterday）**: タスクの編集・完了切替・削除が可能（追加は不可）
- **2日以上前**: 閲覧のみ。編集不可

### 未完了タスクの自動繰り越し

- 0時になったら、前日の未完了タスクを自動的に今日のタスクとしてコピーする
- 繰り越し元のタスクは過去の日付にそのまま残る（履歴として保持）
- API 側で繰り越し処理を行う（クライアントが今日のタスクを取得する際、必要に応じてサーバー側で繰り越しを実行）

### 過去の振り返り

- 日付ナビゲーションで過去の日付に遷移し、その日のタスク達成状況を確認できる

---

## 非機能要件

### UI / UX

- とにかくシンプル。余計な機能・UIを入れない
- 操作が直感的で迷わない
- レスポンスが速い

### データ同期

- iPhone と Mac で同じデータを使う
- バックエンド API を介してデータを共有する

### コスト

- Cloudflare の無料枠で運用する
- 料金が発生する構成は避ける

### 運用

- インフラ管理はシンプルに
- CLI 中心で完結する
- 設定・スキーマ変更は Git 管理する

## 技術構成

### モノレポ構成

```
/apps
  /ios       — iPhone アプリ (Swift / SwiftUI)
  /macos     — Mac アプリ (Swift / SwiftUI)
  /api       — バックエンド API (Cloudflare Workers + Hono + TypeScript)

/packages
  /shared    — 共通の型・スキーマ定義
```

### バックエンド

| 項目 | 選定 |
|------|------|
| ランタイム | Cloudflare Workers |
| フレームワーク | Hono |
| 言語 | TypeScript |
| DB | Cloudflare D1 (SQLite) |
| ファイル保存 | Cloudflare R2（必要になったら） |

### クライアント

| 項目 | 選定 |
|------|------|
| iOS | Swift / SwiftUI (iOS 17.0+) |
| macOS | Swift / SwiftUI (macOS 14.0+) |
| iOS ウィジェット | WidgetKit（大サイズ、ホーム画面用） |

### API エンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| GET | /todos?date=YYYY-MM-DD | 指定日のタスク一覧を取得（今日の場合は自動繰り越し処理も実行） |
| POST | /todos | タスクを追加（今日の日付のみ） |
| PATCH | /todos/:id | タスクの更新（完了切替・並び順変更など。今日と1日前のみ許可） |
| DELETE | /todos/:id | タスクを削除（今日と1日前のみ許可） |

### セキュリティ

- `Authorization: Bearer <secret>` ヘッダで保護
- シークレットは `wrangler secret put` で管理
- ソースコードにシークレットを直書きしない

2. `apps/api` に Workers + Hono をセットアップ
3. D1 作成と初期マイグレーション
4. ToDo CRUD API を実装
5. iOS アプリから API を叩く
6. macOS アプリから API を叩く
7. iOS ウィジェットを追加
